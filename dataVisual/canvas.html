三要素： id: 画布的id width: 画布的宽度 height: 画布的高度

html
<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>

<body>
  <canvas id="canvas1" width="60" height="60">不会显示内容</canvas>
  <script type="text/javascript">
    // 找对象
    let canvas1 = document.querySelector("#canvas1");
    //console.log(canvas1);
    // 获取上下文(画笔)  2d 平面画布  3d 立体画布
    let ctx = canvas1.getContext("2d");
    //let ctx = canvas1.getContext("webgl");
    console.log(ctx);
    // 画矩形 rect()
    ctx.rect(50, 50, 300, 300);
    // 设置画笔样式
    ctx.fillStyle = "aqua";
    // 填充内容
    ctx.fill();
    // 描边渲染路径
    ctx.strokeStyle = "red"; // 设置描边颜色，默认黑色
    ctx.lineWidth = 20; // 设置描边宽度
    ctx.stroke();
  </script>
</body>

</html>

canvas原理 ​

基于 JS 的 2D 绘图 API，通过在 H5 画布上绘制图形来实现。模板骨架中canvas 元素默认被网格覆盖。网格中一个单元相当于 canvas 元素中的一像素。栅格起点为左上角（坐标（0,0））。所有元素的位置都相对于原点定位。所以图中蓝色方形左上角的坐标为距离左边（X 轴）x 像素，距离上边（Y 轴）y 像素（坐标为（x,y））。平移原点到不同的坐标上，旋转网格以及缩放。

canvas和svg区别 ​

名称是否依赖分辨率是否支持事件处理器场景
canvas是，能够以.png 或.jpg 格式保存结果图像否适合图像密集型的游戏，其中的许多对象会被频繁重绘，弱的文本渲染能力
svg否，用 XML 描述 2D 图形的语言是适合带有大型渲染区域的应用程序（地图），复杂度高会减慢渲染速度（过度用 DOM 的应用不快），不适合游戏应用

监测水印div的变化，记录刚生成的div的innerHTML，隔几秒就取一次新的值，一旦发生变化，则重新生成水印。 使用MutationObserver MutationObserver提供了一种能在某个范围内的DOM树发生变化时做出适当反应的能力。 88888888

使用input标签读取用户本地文件后得到的其实是一个Blob对象（更精确的说是File对象，特殊的Blob对象）；而在页面上展示一个图片使用的是img标签；绘制功能用canvas实现。添加水印的功能需要在img标签、canvas画布、Blob对象这三者之间相互转换，通过一些API可以完成这个工作： 我们可以从本地读取图片Blob，然后渲染到img标签，使用canvas绘制img内容并且绘制水印内容到画布，再将canvas内容转为Blob对象上传服务器，这样就完整实现了图片+水印的功能。

本地读取图片得到一个Blob对象，我借助FileReader.readAsDataURL读取Blob的内容，得到一个Base64编码的文件内容，将该内容赋值给img.src从而在浏览器上渲染出本地的图像。当然，img并非必须渲染到DOM树。读取操作是个异步操作，读取完成会触发load事件，便于之后调用，可以用一个Promise包装这个操作，最后返回一个Promise对象。

调用canvas元素画布上下文对象的drawImage方法可实现将img内容绘制到画布。
