  1. src href
名称含义是否阻塞浏览器解析其他文件举例
src引入,替换当前内容是script、img
href引用，建立联系否a、link
  1. script 标签 放在 body底部 或 加 defer async 属性

页面加载完后再加载 JS文件，有助于提高页面加载速度。 ● defer ： 并行加载串行执行，多个 defer 按序执行，一些浏览器可能不是这样。 ● async ： 异步加载，不阻塞页面解析过程，脚本加载完后立即执行脚本，如果文档没解析完会阻塞。多个 async 乱序 ● 动态创建 DOM 对文档加载事件进行监听，当文档加载完成后再动态的创建 script 标签来引入 js 脚本。 ● setTimeout 延迟： 定时器延迟加载 脚本 ● JS 最后加载： 脚本在文档底部，最后加载执行。

没有 defer 或 async ，浏览器读到就开始加载执行，阻塞html加载

名称加载时机执行时机执行顺序优先级
defer都是立即异步并行加载外部 JS 脚本文件，不会阻塞页面的解析延迟执行，等 html 解析完后异步执行，DOMContentLoaded 事件触发执行之前按 HTML 中相对顺序执行脚本较高
async都是立即异步并行加载外部 JS 脚本文件，不会阻塞页面的解析立即异步执行无序：谁先下载完谁先执行高

drag ​

很多拖拽库原理就是这个 vuedragable dragstart：事件主体是被拖放元素，在开始拖放被拖放元素时触发，。 darg：事件主体是被拖放元素，在正在拖放被拖放元素时触发。 dragenter：事件主体是目标元素，在被拖放元素进入某元素时触发。 dragover：事件主体是目标元素，在被拖放在某元素内移动时触发。 dragleave：事件主体是目标元素，在被拖放元素移出目标元素是触发。 drop：事件主体是目标元素，在目标元素完全接受被拖放元素时触发。 dragend：事件主体是被拖放元素，在整个拖放操作结束时触发 source：dragstart()---drag()---dragend() target：dragenter()---dragover()---drop()/dragleave() 1.source设置可拖拽draggable="true" 2.source的ondragstart事件设置被拖拽的数据dataTransfer.setData("text",event.target.id) 3.target的drop事件获取拖拽数据dataTransfer.getData("text") 4.target动态添加dom结点event.target.appendChild(document.getElementById(data));

为什么少用iframe

● iframe创建比其它包括scripts和css的 DOM 元素的创建慢了1-2个数量级。 ● onload事件以及连接池(connection pool)，即iframe会阻塞主页面的Onload事件及iframe和主页面共享连接池，会影响页面的并行加载。 1.iframes阻塞页面加载，影响网页加载速度 ● 及时触发 window 的 onload 事件非常重要。onload 事件触发使浏览器的 “忙” 指示器停止，告诉用户当前网页已经加载完毕。当 onload事件加载延迟后，它给用户的感觉就是这个网页非常慢。 ● window的onload 事件需要在所有 iframe 加载完毕后(包含里面的元素)才会触发，就会影响网页加载速度。通过 JavaScript 动态设置 iframe的SRC可以避免这种阻塞情况。 2.唯一连接池 ● 对每个 web 服务器来说，浏览器只打开极少的几个连接数。老浏览器，每个主机只有2个连接。新浏览器中，连接数增至6个。 ● 大多浏览器中，连接被主页面和它的 iframe共享，有可能iframe中的资源占用了可用连接而阻塞了主页面的资源加载。如果iframe中的内容同等重要，或比主页面更重要，这很好。然而通常情况下iframe中内容不太重要，iframe 占用连接数是不可取的。解决方案是在优先级更高的资源下载完成后再动态的给iframe的src赋值。 ● iframe影响，目前框架的优点可以使用Ajax实现，这在某种角度也是一种替代方案。

场景

iframe是独立区域，不受 parent的CSS或全局JavaScript影响。 1.网页编辑器 2.跨域通信 3.历史记录管理，解决ajax化网站响应浏览器前进后退按钮的方案，在html5的history api不可用时作为一种替代； 4.纯前端的utf8和gbk编码互转。比如在utf8页面需要生成一个gbk的encodeURIComponent字符串，可以通过页面加载一个gbk的iframe，然后主页面与子页面通信的方式实现转换； 5.iframe实现无刷新文件上传，FormData不可用时作为替代方案； 6.创建全新的独立的宿主环境。隔离或者访问原始接口及对象，比如有些前端安全的防范会覆盖一些原生的方法防止恶意调用，通过创建一个iframe，然后从iframe中取回原始对象和方法来破解这种防范； 7.加载广告，例如联盟广告；
