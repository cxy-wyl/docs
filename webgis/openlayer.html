点扩散 ​

获取dom元素配合css动画 背景：海量点，给point加扩散动效

方案1：定时器每隔1s执行回调改变样式，扩散改变radius和optical, 流动改变位移，

问题：时间不准，内存泄漏（可能定时器没清除，定时器对回调引用，回调函数中引用的对象也不会被释放），频繁触发占用过多的CPU资源，从而影响其他任务的执行。由于JavaScript是单线程的，所以定时器的实现是在当前任务队列完成后再执行定时器的回调的，假如当前队列任务执行时间大于定时器设置的延迟时间，定时器就不可靠了

方案2：监听地图渲染事件，不断改变样式触发回调，

问题：会互斥，正在执行回调，下一秒显示另一个图层，也会触发渲染事件，视觉上看到就会卡一帧，而且样式很丑，显示格网和扩散都会触发渲染事件。可能内存泄漏要手动移除监听。

方案3：gif图，矢量图层添加一个点，赋予 ol.style.Icon 样式，将图片的 src属性赋为一张 GIF 图，设置icon位置角度

问题：openlayers用icon功能, 将gif图标导入要素, 未生效, 本以为会出现动态的图标，结果发现并没有，动态图片变成静态图片。查相关资料发现openlayers不支持gif动态图标(100%)，然后引入第三方库gifler.js, 封装giflerFunction方法, 注入全局, 重新创建canvas画布引入地图要素, 替代图标实现水波纹动画效果(100%)。性能: 模拟5000个地图点数据, 初始化地图加载严重卡顿延迟, 发现此方案不行, gif动图太大导致渲染卡顿(100%)

方案4(扩散)：用 CSS3 动画，依附于 HTML 元素，OpenLayers实现 Popup 时，Popup 就是 HTML 元素，canvas图标点击出现弹框

解决：CSS3纯样式实现动画, dom元素使用3层光圈定位, 模糊阴影, 2S动画帧效果实现, 定位在循环创建要素点时, 生成跟点数量一直的dom元素, 使用openlayers Overlay类替代要素点, 通过原生js将html字符元素嵌入Overlay类中实现(100%)，性能: 模拟5000和10000个地图点数据, 初始化地图加载流畅(100%)动画效果和地图渲染分离，还能自定义样式，尽量用 CSS3 的 transform，因为 transform 属性不会改变元素的布局

聚合效果 ​

● 画布数据源不是多个点，而是一个聚合类这样比原始的点少很多，设distance表示两点距离为distance时将被聚在一起，从source移除放入聚合层，minDistance聚合块间最小像素距离 ● 传distance，像素间距离为distance，就push到聚合[]，minDistance是两个聚类中心的距离，如果小于就合并 ● 将地图上的所有点作为待处理的点集合，随机选个起始点作为第一个聚类中心，也可以根据某种启发式规则选择的。 ● 计算剩余待处理的点集合中的每个点，当前已有聚类中心的距离。小于distance就push,否则新建新的聚类中心。可用不同的距离度量方法，如欧氏距离、曼哈顿距离等。 ● 每次处理完更新每个聚类中心位置，计算聚类中所有点的平均位置来得到新的中心位置。不断重复处理待处理的点集合，直到第一次聚合完成。 ● 放大后每个聚类越来越少，如果各自聚类中心中某个点距离原有聚类中心大于distance，则分离出去成为一个新的聚类中心 ● 缩小后每个聚类越来越多，如果某两个聚类距离小于minDistance，则合并更新中心位置，删除空聚类 ● 定义2种样式，每个点的样式设置的时候判断它所在聚合类的数量，如果是1就显示具体样式icon，不是1就显示聚合样式圆，上边text为聚合类的size ● 不一定得到全局最优解，只是局部最优算法。因此，聚合中，贪心可能得到不同的结果，具体取决于起始点的选择和贪心策略设计。 ● 问题在于：如果两个点很近，最小1px ● 优化：根据密度自适应阈值进行贪心算法

//贪心
const threshold动态阈值 = getAdaptiveThreshold(dataSet待处理的数据集, neighborhoodRadius阈值内点认为是同一聚类，半径，外部传的distance);//动态计算每个聚合点的阈值distance，小于distance分配给距离最近的聚类中心，大于distance新建聚类，更新聚类中心
//根据给定的数据集和邻域半径计算出一个自适应阈值
function getAdaptiveThreshold(dataSet, neighborhoodRadius) {  
  let densities = [];  //存储每个数据点的密度值
  for (let i = 0; i < dataSet.length; i++) {  
    densities.push(calculateDensity(dataSet, i, neighborhoodRadius));  //将当前数据点（dataSet[i]）和邻域半径传给该函数，将返回的密度值加到densities数组中。///相当于densities.push(2)
  }  
  // 计算密度的平均值或中位数作为自适应阈值  
  const threshold = calculateThreshold(densities);  
  return threshold;  
}
//返回每个点的密度值
function calculateDensity(dataSet, pointIndex, neighborhoodRadius) {  
  let count = 0;  //计算在给定邻域内的点数。
  for (let i = 0; i < dataSet.length; i++) {  
      //聚合内部两点距离小于外部传的distance
    if (distance(dataSet[pointIndex], dataSet[i]) <= neighborhoodRadius) {  
      count++;  //邻域内的点数加一
    }  
  }  
  return count;  //代替初始外部传的distance
}

// 计算距离
    getDistance(lat1, lng1, lat2, lng2) {
        let rad1 = lat1 * Math.PI / 180.0;
        let rad2 = lat2 * Math.PI / 180.0;
        let a = rad1 - rad2;
        let b = lng1 * Math.PI / 180.0 - lng2 * Math.PI / 180.0;
        let r = 6378137;
        let distance = r * 2 * Math.asin(Math.sqrt(Math.pow(Math.sin(a / 2), 2) + Math.cos(rad1) * Math.cos(rad2) * Math.pow(Math.sin(b / 2), 2)))
        distance = (distance / 1000).toFixed(2);
        return `${distance} km`;
    }

// getTurfArcFeature
    getTurfArcFeature(start, end) {
        let turfFormat = new GeoJSON();
        let line = turf.lineString([
            start,
            [start[0] + (end[0] - start[0]) * 0.5, start[1] + (end[1] - start[1]) * 0.65],
            end
        ]);
        let curved = turf.bezierSpline(line);
        let length = turf.length(curved, { units: 'meters' });
        let bF = turfFormat.readFeature(curved);
        bF.set("length", length);
        return bF;
    }
